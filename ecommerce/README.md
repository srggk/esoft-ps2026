# Ecommerce Project (React Lessons)

##  Итерации разработки
1. СТАТИЧЕСКАЯ ВИТРИНА ТЕЛЕВИЗОРОВ

    Цель: создать базовую статическую страницу каталога телевизоров. Никакой интерактивности — только вывод данных и вёрстка.

    Макет: https://www.figma.com/design/kqIZeoJ07t8ymMvjR0jLEM/Home-Assignment-2026?node-id=20-496&t=kPWWqMnOEayuhd1Z-0

    * ☑ Страница рендерит 8 карточек телевизоров
    * ☑ Карусель изображений в карточке: стрелки при наведении, индикатор количества фото, смена фото по клику
    * ☑ Бейдж «Special Offer» отображается на товарах с isSpecialOffer === true
    * ☑ В карточке: бренд, модель (line-clamp 2 строки), цена в формате $X,XXX
    * ☑ Кнопка избранного (сердечко) с переключением состояния (локально)
    * ☑ Кнопка «Add to Cart» работает как счётчик: «−» / «N in cart» / «+», возврат к исходной кнопке при 0
    * ☑ Header: логотип «TechStore» (ссылка на /tv), три таба категорий, иконка корзины, иконка пользователя
    * ☑ Sidebar: dropdown брендов, поля Min/Max цены, кнопка «Apply Filters» — визуально присутствуют, фильтрация не работает
    * ☑ Баннер «Special Deal» с таймером 0:59:59 (статичное время)
    * ☑ Footer: четыре колонки (About, Support, Legal, Newsletter) + копирайт
    * ☑ Кнопки категорий в Header не переключают контент (рендерятся только TV)
    * ☑ Sticky header
    * ☑ Footer прижат к низу страницы
    * ☑ Вёрстка адаптивная (1 → 2 → 3 → 4 колонки)

2. УЛУЧШЕННАЯ ВИТРИНА МАГАЗИНА ЭЛЕКТРОНИКИ

    Цель: научиться управлять состоянием приложения, передавать состояние от родителя к потомку.

    Макет: https://www.figma.com/design/kqIZeoJ07t8ymMvjR0jLEM/Home-Assignment-2026?node-id=510-1113&t=T6X0wtKq4ULp8Qwk-4

    Глобальное состояние:
    * ☑ Приложение обёрнуто в Container, хранящий pageType и cart
    * ☑ Container передаёт состояние и сеттеры пропсами в Header и Content
    * ☑ Content рендерит нужную страницу в зависимости от pageType

    Переключение категорий:
    * ☑ Кнопки TV / Phone / Laptop в Header переключают содержимое витрины
    * ☑ Логотип «TechStore» переключает на TV
    * ☑ При смене категории: фильтры сбрасываются, сортировка сбрасывается
    * ☑ При смене категории корзина НЕ меняется

    Витрины (TvListing / PhoneListing / LaptopListing -> merged to Listing):
    * ☑ TvListing показывает товары с category === 'tv' (20 -> 8 товаров)
    * ☑ PhoneListing показывает товары с category === 'phone' (8 товаров)
    * ☑ LaptopListing показывает товары с category === 'laptop' (8 товаров)

    Фильтры:
    * ☑ Dropdown брендов: список брендов вычисляется из данных текущей категории
    * ☑ Значение по умолчанию — All Brands
    * ☑ Поля Min Price (пустое) и Max Price (5000) — управляемые
    * ☑ Листинг обновляется ТОЛЬКО при нажатии «Apply Filters»
    * ☑ Фильтрация корректно работает по бренду и диапазону цен

    Сортировка:
    * ☑ `<select>` с опциями Price: Low to High / Price: High to Low
    * ☑ Значение по умолчанию — Low to High
    * ☑ Счётчик «N products» показывает количество товаров с учётом фильтров
    * ☑ Сортировка корректно упорядочивает товары по цене

    Корзина:
    * ☑ Иконка корзины в Header с бейджем (общее количество товаров)
    * ☑ При клике на корзину — переход на страницу корзины
    * ☑ В корзине: список товаров, +/- кнопки изменения количества, кнопка удаления
    * ☑ Сводка заказа: Subtotal, Tax (8%), Shipping, Total
    * ☑ «Proceed to Checkout» — кнопка есть, но пока ничего не делает
    * ☑ «Back to Shopping» — возвращает на витрину TV

    Карточка товара:
    * ☑ Add to Cart теперь меняет глобальный стейт корзины (а не локальный)
    * ☑ Кнопка-счётчик (− / N in cart / +) работает с глобальной корзиной

    Бонусы:
    * ☑ Корзина сохраняется в sessionStorage между перезагрузками

3. УЛУЧШЕННАЯ ВИТРИНА МАГАЗИНА ЭЛЕКТРОНИКИ

    Цель: научиться работать с побочными эффектами (useEffect, интервальные таймеры), управлять подписками и очисткой, работать с внешними API (fetch, AbortController, состояния loading/success/error), создавать модальные окна.

    Живой таймер:
    * ☑ При монтировании компонента отображается время "0:59:59"
    * ☑ Обратный отсчёт начинается автоматически сразу после монтирования
    * ☑ Формат времени — H:MM:SS (одна цифра часов, две цифры минут, две цифры секунд)
    * ☑ Кнопка "стоп" останавливает обратный отсчёт, текст кнопки меняется на "возобновить"
    * ☑ Кнопка "возобновить" продолжает обратный отсчёт с текущего значения, текст кнопки меняется на "стоп"
    * ☑ При остановке таймера интервал очищен (нет утечки памяти)
    * ☑ Кнопка "рестарт" сбрасывает время на "0:59:59"
    * ☑ Если таймер был остановлен — после рестарта отсчёт остаётся на паузе
    * ☑ Если таймер шёл — после рестарта отсчёт продолжается с "0:59:59"
    * ☑ При достижении "0:00:00" отсчёт прекращается, время заменяется на текст "таймер истёк"
    * ☑ При достижении "0:00:00" кнопка "стоп / возобновить" заблокирована (disabled)
    * ☑ При достижении "0:00:00" кнопка "рестарт" активна и визуально подсвечена
    * ☑ Из состояния "таймер истёк" нажатие "рестарт" устанавливает "0:59:59" и сразу запускает отсчёт
    * ☑ Кнопка "крестик" удаляет блок таймера со страницы (компонент размонтируется)
    * ☑ При удалении компонента интервал очищен (нет утечки памяти)

    Виджет погоды (с помощью открытого API open-meteo.com)

    > API-ключ в запрос добавляется, но сервис его игнорирует (сделано для демонстрации работы с .env), потому что тяжело найти API сервисы погоды с ключом, которые работают в РФ без ***

    * Первый показ:
        * ☑ При монтировании виджета показывается сцена загрузки (скелетон/лоадер)
        * ☑ Выполняется запрос в Geocoding API с городом "Тюмень"
        * ☑ Если Geocoding вернул отказ — основная сцена: пустой инпут, пустая область погоды, предупреждение "Не удалось получить данные для города Тюмень" под инпутом
        * ☑ Если координаты получены — выполняется запрос погоды по координатам
        * ☑ Если запрос погоды вернул ошибку — основная сцена: пустая область погоды, "Тюмень" в инпуте, сообщение "Не удалось получить данные" под областью погоды
        * ☑ Сообщение "Не удалось получить данные" висит до получения результатов следующего запроса
    * Успешная загрузка:
        * ☑ Основная сцена: область погоды заполнена, имя города в инпуте
        * ☑ Инпут доступен для редактирования
        * ☑ Кнопка "получить погоду" присутствует рядом с инпутом
    * Ручной ввод города:
        * ☑ Пользователь вводит название города и нажимает "получить погоду"
        * ☑ На время выполнения запросов инпут и кнопка неактивны (disabled)
        * ☑ Выполняется запрос в Geocoding API по введённому имени
        * ☑ Если координат нет — инпут очищается, под инпутом сообщение "Не удалось получить данные для города <введённое значение>", ранее отображённая погода остаётся на экране
        * ☑ Сообщение под инпутом висит, пока пользователь не изменит текст
        * ☑ Компонент запоминает города с неудачными запросами и показывает предупреждение при повторном вводе того же названия
        * ☑ Если координаты есть — выполняется запрос за погодой
        * ☑ Если запрос за погодой не выполнился — основная сцена: пустая область погоды, имя города в инпуте, сообщение "Не удалось получить данные" под областью погоды, висит до следующего запроса
    * Общие:
        * ☑ При размонтировании компонента во время запроса — запрос отменяется через AbortController
        * ☑ API-ключ не захардкожен, берётся из переменной окружения (import.meta.env)
        * ☑ Файл .env добавлен в .gitignore
        * ☑ Файл .env.example присутствует в корне проекта и не игнорируется git
        * ☑ Кнопка "крестик" удаляет виджет со страницы (размонтирование, отмена запросов)
    * Бонус:
        * ☑ При монтировании виджета выполняется запрос в navigator.geolocation API
        * ☑ Если пользователь разрешил геолокацию — погода определяется по координатам пользователя
        * ☑ Если пользователь отказал — fallback на город по умолчанию (Тюмень)

    Кнопка с модальным окном:
    * ☑ На странице есть блок (кнопка), при нажатии на который открывается модальное окно (кнопка пользователя в header)
    * ☑ Модальное окно реализовано через ReactDOM.createPortal (рендерится вне основного DOM-дерева приложения)
    * ☑ В модальном окне есть кнопка закрытия — при нажатии окно закрывается
    * ☑ Модальное окно закрывается при нажатии клавиши Esc
    * ☑ У модального окна есть оверлей, визуально оттеняющий окно от заднего фона
    * ☑ Стилизация модального окна и оверлея произвольная, но окно визуально отделено от фона
    * ☑ Содержимое модального окна (кроме кнопки закрытия) — форма входа в аккаунт
